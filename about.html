About this site

b-log 是一個雙重用途的內容系統，完全存在於 GitHub 儲存庫中。它既是分享見解的部落格，也是管理項目的待辦清單。每篇文章都是一個 Markdown 檔案，一個在 data/posts.json 中的小型條目，僅此而已。

Why it works well with AI

  • Prompts are simple: "Create a Markdown file in content/posts and update the catalog."
  • No build step or tooling - GitHub Pages publishes static files as-is.
  • Structure stays consistent, so AI agents can extend or refactor without fear.

How to publish

  1. Duplicate an entry in data/posts.json with a new slug.
  2. Create a Markdown file with the same slug inside content/posts/.
  3. Commit, push, and GitHub Pages does the rest.

Built on GitHub Pages with a sprinkle of JavaScript.
