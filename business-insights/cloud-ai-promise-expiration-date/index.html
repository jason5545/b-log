雲端 AI 的承諾有保存期限

By Jason ChienPublished Apr 21, 20263 min
GitHub CopilotAnthropicClaude訂閱制本地模型商業觀察AI 工具agentic

我離開 claude.ai 是 4 月初的事。理由不複雜：一連串的政策問題讓我意識到，付費兩年換來的，不是一個穩定的工具，而是一個隨時可以改變條款的服務。

當時的過渡方案是 GitHub Copilot Pro+，透過它存取 Claude Opus。這個方案在邏輯上是合理的——換一個中間層，但模型本身沒變。

昨天（4/20）GitHub 官方宣布：Opus 從 Pro 完全移除，Pro+ 僅保留 Opus 4.7，舊版本全數退場。同一天，Pro、Pro+、Student 三個方案全面暫停新使用者註冊——理由是要優先保障現有付費使用者的服務品質。

這個決定有其商業邏輯。洩漏的內部文件顯示，Copilot 每週的運營成本從今年年初到現在已經翻倍。GitHub 副總裁 Joe Binder 在公告裡的措辭相當直白：「長時間運行的平行 agentic 工作流，所消耗的資源遠超過原本方案結構的設計上限。」換句話說，有些使用者單一請求的成本已經超過他們整個月的訂閱費。Microsoft 內部也已在規劃長期轉向按 token 計費的機制。

我不是要批評 GitHub 的決定。真正值得觀察的是這件事背後的結構性問題。

雲端 AI 的本質是訂閱租賃，不是工具購買

你租的不是模型能力，你租的是「今天這個版本的存取權」。條款可以改，模型可以換，功能可以鎖在更貴的方案裡。這不是哪家公司特別惡意，這是訂閱制商業模式的內建邏輯。

過去幾個月，我觀察到的節奏是這樣的：先以低價或免費擴大使用者依賴，等工作流程建立起來、替換成本夠高之後，再開始調整條款。Copilot 對學生方案的操作就是一個教科書案例——先給 Opus，讓使用者在最關鍵的學習期建立習慣，再以「永續性」為由移除。

從 claude.ai 到 GitHub Copilot，這半年來我親歷的劇本如出一轍：

  • Anthropic：prompt cache bug 兩週不修、system prompt 差別計費、使用規範模糊化
  • GitHub：免費試用暫停、Opus 移除、新使用者註冊關閉、按 token 計費浮上檯面

不同公司，相同邏輯。

出口只有一個

對個人開發者來說，這個遊戲的出口只有一個：降低對任何單一雲端服務的依賴深度。

本地模型在兩年前還是一個「有理想但不實用」的選項。現在 70B 量化模型在消費級硬體上已經可以執行，品質差距在縮小，而雲端服務的穩定性預期也在下降。這兩條線正在交叉。

我自己的結論很直接：等手邊的硬體到位，主力工作流會移到本地。不是因為本地模型現在比雲端好，而是因為它不會在某個週一早上突然告訴你「這個功能已移至更高方案」。

工具的價值不只來自能力，也來自可預期性。這是雲端 AI 目前最脆弱的地方。
